## Releases

- Follow the Pride versioning scheme (`PROUD.DEFAULT.SHAME`) described in [`RELEASE_TEMPLATE.md`](./RELEASE_TEMPLATE.md).
- When cutting a release, copy the template sections (Highlights/Added/Changed/Fixed/Known Issues/Checks) and fill them with links to issues and PRs.
- Attach relevant artifacts (web bundle, desktop build, API image) and media (screenshots, GIFs).
